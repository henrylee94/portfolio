Scalp 分析器

一个用 Python 写的美股 scalp 实时分析器,对着本地 Moomoo OpenD 实例跑,yfinance 做兜底数据源。它拉 1m/5m/15m K 线,用 pandas + ta 算 EMA/MACD/RSI 和以当日开盘锚定的 VWAP,检查七种经典形态——开盘区间突破、VWAP 反弹、bull flag——配一张 confluence 清单和一个 0–10 的安全分,午间震荡和动能衰减都会扣分。包装成了 Claude Code skill,一句「check NVDA」就返回一张 entry/target/stop 卡片。pipeline 刻意做成线性——feed → candles → checks → score → card——读数不对时,要查的环节只会有一个。只做分析——每一笔订单都是手动下的。

  • Python
  • pandas + ta
  • Moomoo OpenD
  • yfinance 兜底
  • Claude Code Skill

股票哨兵

我最大的个人项目:一个住在 Telegram 里的 AI 交易助手——前面是 python-telegram-bot,后面是 OpenAI API 驱动的多 agent pipeline——分析器、技术分析、策略生成、回测、最终决策——行情数据来自 Finnhub 和 yfinance,配一个 Streamlit 后台,部署在 Zeabur 上。agent 按固定链条跑——上一个的输出就是下一个的输入——所以每个结论都能沿着产生它的回测、策略、分析各阶段追回去。策略放在一个 JSON skill 库里:orchestrator 按市场状态匹配策略,按跟踪到的表现排序,再把交易结果喂回去。每个回答都被强制成紧凑格式:结论、entry、target、stop、一条理由。约 8k 行 Python。

  • python-telegram-bot
  • OpenAI API
  • Finnhub
  • Streamlit · Zeabur

AI 家用服务器

已经不是蓝图了——一台 Minisforum UM890 Pro(Ryzen 9 8945HS,32GB)现在 24/7 跑在我的客厅里:Windows 11 加 WSL2 Ubuntu,Hermes 裸装在里面,外加一个 FastAPI 交易服务和一个静态每日新闻页。访问只走 Tailscale——零开放端口,由 Windows portproxy 桥进 WSL。诚实的教训:本地模型本来是计划,但它们输了——Ollama 跑 Qwen 2.5 7B 和 Llama 3.1 8B,纯 CPU 推理跟不上多轮 tool calling,所以现在由一个分层 router 把日常工作发给便宜的云端模型,把重型模型留给深度推理。安全模型倒是完好地经受住了现实:沙箱里的 agent、出站白名单、每个服务一个低权限账号,以及一张绿/黄/红委派矩阵——银行凭证永远是红,永远。

  • UM890 · WSL2
  • 仅 Tailscale 可达
  • Ollama → 分层云端
  • 24/7,自 2026 年 5 月

个人财务规划系统

一个我今年做来算自己那本账的双语(EN / 中文)财务规划应用:三个生活档位的长期财务推演、提前还房贷的数学、带十年推演的资产配置,外加一个现金与持仓追踪器。所有数据都存在 localStorage,支持 JSON 导出——没有账号,没有服务器。Vanilla JavaScript,不用框架——给一个用户用的工具,就该简单到一个晚上能维护完。
