Cross-stack mission pipeline — three languages, exactly-once

Spot users close to a reward, give them a 7-day challenge, deliver instantly — and never twice.

4 services · 3 languages exactly-once, layered 0 legacy changes pre-prod verified

Situation

Marketing wanted automatic, time-boxed missions for users close to their next loyalty tier: detect them in real time, run a 7-day window, grant the upgrade instantly on success.

Constraint

The reward path is money-adjacent: a duplicate grant is a real loss, and events arrive at least-once from both the stream and the workflow engine — two independent sources of duplicates.

Approach

Built the chain end-to-end across four services in three languages — the boundaries followed the platform's existing seams (the streaming platform, the workflow engine, the reward facade), composing proven engines instead of building new ones: a Java/Flink job filters the event stream and computes entry-anchored goals from a lookup table; a Rust + Temporal engine runs one durable workflow per user; a NestJS facade grants through the existing upgrade path — zero changes to legacy systems. Exactly-once is layered: an atomic claims ledger (one chance per user, tier, month) gates enrollment, and a human-readable idempotency key (`mission:user:tier:month`) gates the grant — debuggable at a glance in production. Thresholds live in exactly one place — producers broadcast unconditionally, so two repositories can never disagree about the same number.

Outcome

Verified end-to-end against real traffic in the pre-production lane; funnel metrics at every stage. The architecture — detect / orchestrate / grant with single-owner config — is the template for the next mission types.
