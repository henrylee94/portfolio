02大局观

系统,从高处看

超越任何单个功能——拆解、平台,以及运行它的纪律。

01

把老单体拆成 gRPC 微服务。

是增长逼出来的重构——每天数百万笔交易、峰值 ~1,000/s 还在往上爬,老架构撑不住了。我带队把平台最关键的流程迁到新的 NestJS/gRPC + Temporal 栈:框架先行、迁移途中清理死码、切流由 SDK 承载并做 endpoint 级降级回退。迁移即现代化,每一步都内建回退。

02

搭起团队据以交付的契约先行平台。

proto-first、契约先行:一份 .proto 生成 SDK 与所有服务共享的 gRPC 类型,接入就是一个 import、零样板。新增一个请求类型,从要人翻代码、等 merge 的 30–60 分钟,变成 两分钟自助表单——PR、发版、SemVer、changelog、通知。100+ 已发布版本;200+ 请求类型跑在一个 base workflow 上。这是 developer enablement,不只是一个库。

03

用生产级可观测性把它跑起来。

我每天早上盯的:队列积压、workflow 重试、DB 负载、慢查询、CPU/内存尖峰与泄漏——都在 Grafana 上,配合阿里云 SLS 做基于日志的告警与线上排查,底层是 OpenTelemetry 链路和 Prometheus 指标。优雅停机是不可妥协的 readiness gate——它是从一次真实事故里长出来的规则。先索引,后缓存:当年查一天数据都超时的查询,如今随便查 30 天窗口。

03我怎么工作

我怎么工作

先讲原则,再谈任务。系统只是工作的一半——另一半是把做系统的人带出来。

P1

先给风险定价,再给工作量定价。

第一个问题从来不是「要做多久」,而是「这东西错了会坏什么,谁会感觉到?」失败成本决定工程强度。

L1

培养 owner,不是接任务的人。

每个工程师端到端 own 一个域——连和业务谈需求都包括在内。

L2

先指挥事故,再让它交房租。

先 rollback,再排查——然后让事故交出回报:一份 playbook、一条发版规则、一份向上同步的报告。我们的风控告警 pipeline,就是从一次漏扣费里长出来的。

为下一次变更设计,而不是为这次交付。

业务以后会想调的部分做成配置,而不是代码——一个新需求变成一次运维改动,而不是一次 deploy。

按 ownership 拆解,而不是按 ticket。

写代码之前契约先拆好——一个横切五个仓库的功能,曾在一天内落地。

用证据做决定,带着 rollback 去执行。

大的选型要 benchmark 和 POC——一次 OLAP 选型,三个引擎各跑 30 条查询 × 50 轮才拍板。

把工作变成杠杆。

任何做过两次的事都要变成工具——集成 SDK、事故 playbook、全队都在用的 AI 诊断 bot。

校准挑战的难度。

要有拉伸,但不能淹死——我看他们卡在哪就点一下,不抢键盘。

搭配性格,不只搭配技能。

一个细致但慢的人,配一个推得快但容易糙的人——像调任何系统一样调搭配,漂了就再调。

留一个机动位。

一个工程师,哪里压力大就去哪里——我自己就是在这个位置上长大的,所以特意把它做进了团队。

用一个 deadline 换后面十个。

地基打一次,后面十个需求就都变成配置——每个「不」都带着理由和替代方案一起出口。
