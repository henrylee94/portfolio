02at altitude

Systems, at altitude

Beyond any single feature — the decomposition, the platform, and the discipline to run it.

01

Decomposed a legacy monolith into gRPC microservices.

Growth forced the rewrite — millions of transactions a day, ~1,000/s and still climbing past what the old architecture could hold. I led the migration of the platform's most critical flows onto a new NestJS/gRPC + Temporal stack: framework first, dead code retired on the way, the cutover carried by the SDK with endpoint-level fallback. Migration as modernization, with a rollback built into every step.

02

Built the contract-first platform the team ships on.

Proto-first by contract: one .proto generates the gRPC types the SDK and every service share, so integration is one import with zero boilerplate. Adding a new request type went from a 30–60-minute, person-dependent process to a two-minute self-serve form — PR, publish, SemVer, changelog, notify. 100+ published versions; 200+ request types run through one base workflow. Developer enablement, not just a library.

03

Ran it with production-grade observability.

What I watch every morning: queue backlog, workflow retries, DB load, slow queries, CPU/memory spikes and leaks — on Grafana, with Aliyun SLS for log-based alerting and incident triage, OpenTelemetry traces and Prometheus metrics underneath. Graceful shutdown is a non-negotiable readiness gate — it came out of a real incident. Index first, cache second: queries that timed out on a single day of data now scan a 30-day window.

03how i work

How I Work

Principles first, tasks second. The systems are half the job — the other half is building the people who build them.

P1

Price the risk before pricing the work.

The first question is never "how long will it take" — it's "what breaks if this is wrong, and who feels it?" Failure cost decides engineering rigor.

L1

Grow owners, not task-takers.

Each engineer owns a domain end-to-end — requirements negotiation with business included.

L2

Command the incident, then make it pay rent.

Roll back first, diagnose second — then the incident pays: a playbook, a release rule, a report upward. Our risk-alerting pipeline was born from a single missed charge.

Design for the next change, not this delivery.

What business will tune later becomes configuration, not code — a feature request becomes an ops change, not a deploy.

Decompose by ownership, not by ticket.

Contracts split before anyone writes code — a cross-cutting feature once landed across five repositories in one day.

Decide with evidence, commit with a rollback.

Big choices get benchmarks and POCs — one OLAP evaluation ran 30 queries × 50 rounds across three engines before we committed.

Turn work into leverage.

Anything done twice becomes a tool — integration SDKs, incident playbooks, an AI diagnosis bot the whole team runs.

Calibrate the challenge.

Stretch without drowning — I watch where people stall and nudge; I don't grab the keyboard.

Pair temperaments, not just skills.

A meticulous slow-mover next to a fast pusher who gets careless — tuned like any other system, re-tuned when they drift.

Keep a floating seat.

One engineer goes wherever the pressure is — that seat is how I grew up, so I built it into the team on purpose.

Trade one deadline for the next ten.

Build the base once and the next ten requests become configuration — a "no" always ships with its reason and its alternative.
